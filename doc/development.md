## Technical Implementation Details

### 1. Script Injection

Typora is Electron-based; its editor runs JS from `index.html`. Appending a `<script>` tag runs custom code in Typora's renderer process.

**Install location**: `/Applications/Typora.app/Contents/Resources/TypeMark/index.html`

**Injection** (`bin/install.sh`):
```html
<script src="./appsrc/main.js" aria-hidden="true" defer></script>
<script src="./ai-edit/typora-ai-edit.js" defer></script>   ← inserted line
```

**Plugin file**: `/Applications/Typora.app/Contents/Resources/TypeMark/ai-edit/typora-ai-edit.js`

### 2. Modular Architecture

The plugin source is split into 13 modules under `src/modules/`. The build script `build.sh` concatenates them in order, wraps in an IIFE, and outputs `src/typora-ai-edit.js`.

```bash
bash build.sh        # Concatenate modules → src/typora-ai-edit.js
```

All modules share the same function scope (IIFE), so variables and functions defined in earlier modules are accessible in later ones. Module order matters for dependency resolution.

### 3. Configuration (`02-config.js`)

**Storage**: `localStorage` (key: `typora-ai-edit-config`)

**Data shape**:
```javascript
{
  provider: "chatgpt" | "openai_compat",
  model: "gpt-5.4",
  web_search: false,
  openai_compat: { base_url, api_key, models_input, models: [{name, available, web_search, vision}] },
  models: ["gpt-5.4", "gpt-5.4-mini", ...],
  prompts: {
    optimize: { system, user },
    optimize_with_context: { system, user },
    describe_image: { system, user },
    qa: { system, user },
    qa_with_context: { system, user }
  },
  shortcuts: { qa: { key, metaKey, shiftKey, ctrlKey, altKey } }
}
```

**Load strategy**: `loadConfig()` merges saved config with `DEFAULT_CONFIG` so new fields get defaults. Default prompts are language-aware (Chinese or English based on `isChinese`).

### 4. OAuth Token Loading (`03-platform.js`)

**Token path**: `~/Library/Application Support/oauth-cli-kit/auth/codex.json`

**Home directory resolution** (`getHomePath()`, 5-level fallback):
1. `process.env.HOME`
2. `window._options.appDataPath` → regex extract
3. `window._options.userPath` / `homePath`
4. `bridge.callSync("controller.runCommandSync", "whoami")`
5. `window.dirname` / `__dirname` → regex extract

**File read** (`readFileContent()`, 3 methods):
1. `window.bridge.callSync("path.readText", path)` — macOS Typora
2. `window.reqnode("fs").readFileSync(path)` — Windows/Linux Typora
3. `require("fs").readFileSync(path)` — generic Node.js

### 5. API Calls (`04-api.js`)

**Two providers**:

| | ChatGPT Codex | OpenAI Compatible |
|--|--|--|
| Endpoint | `POST chatgpt.com/backend-api/codex/responses` | `POST {base_url}/chat/completions` |
| Auth | `Bearer <oauth_token>` + `chatgpt-account-id` | `Bearer <api_key>` |
| Request format | `instructions` + `input[]` array | `messages[]` array |
| Image format | `{ type: "input_image", image_url }` | `{ type: "image_url", image_url: { url } }` |
| Web search | `tools: [{ type: "web_search" }]` | `tools: [{ type: "function", function: { name: "web_search" } }]` |

**Unified entry**: `callAPI()` dispatches to the correct provider based on `config.provider`.

**SSE parsing**: Two parsers — `parseSSE()` for Codex (events split by `\n\n`, type `response.output_text.delta`) and `parseOpenAISSE()` for OpenAI (events split by `\n`, standard `choices[0].delta.content`).

**Abort**: Global `currentAbort` (`AbortController`), called by `abortCurrentRequest()`.

### 6. Model Testing (`05-model-test.js`)

`testOpenAIModels(baseUrl, apiKey, modelNames, onLog)`:
1. Fetch `/models` endpoint to get available model list
2. For each model: test availability (`/chat/completions` with `max_tokens: 1`)
3. Test web search support (send request with `tools` parameter)
4. Test vision support (send request with `image_url` content)
5. Return array of `{ name, available, web_search, vision }`

Auto-fetch: when `modelNames` is empty, fetches from `/v1/models` automatically.

### 7. Editor Interaction (`06-editor.js`)

**Normal text**:
- Selection: `window.getSelection().toString()`
- Save: `sel.getRangeAt(0).cloneRange()`
- Replace: `document.execCommand("insertText", false, newText)`

**CodeMirror code blocks** (Mermaid, fenced code):
- Detection: `findFocusedBlock()` — checks `.md-fences.md-focus`, `document.activeElement`, `.CodeMirror-focused`
- Read: `cm.getValue()` (full block) or `cm.getSelection()` (selected text)
- Write: `cm.setValue(newText)` or `cm.replaceSelection(newText)`
- Re-activation: `reactivateBlockCM(blockEl)` — simulates click on block element to re-create stale CM instances after dialog closes

**Rendered HTML blocks** (SVG, HTML):
- Source extraction: `getHtmlBlockSource()` — parses document Markdown to find matching `<tag>...</tag>` block
- Replacement: `replaceInMarkdown(oldSource, newSource)` — reads full Markdown, does string replace, writes back via `fs.writeFileSync`, triggers Typora reload via `File.reloadContent()` / `File.editor.reload()` / `location.reload()`

**Q&A insertion**: `insertQAResponse(text, targetEl)` — saves cursor target before dialog opens, wraps text in blockquote (`> `), focuses `#write`, sets range after target element, inserts via `execCommand`.

### 8. Image Processing (`03-platform.js`)

**Data URL extraction** (`getImageDataUrl()`):
1. `data:image/…` → use directly
2. `http(s)://…` → canvas export or Node.js download (`fetchImageAsDataUrl`)
3. Local file → `readFileAsBase64()` + MIME detection
4. Canvas fallback → `drawImage` + `toDataURL`

**Compression** (`compressImageDataUrl()`): Resize to max 2048px (long edge), convert to JPEG at 80% quality via canvas. Logs original vs compressed size.

**Node.js download** (`fetchImageAsDataUrl()`): Uses `http`/`https` modules via `window.reqnode`, handles redirects, converts response buffer to base64 data URL.

### 9. UI: Context Menu (`08-ui-menu.js`)

**Hook**: Capture-phase `contextmenu` listener on `#write` and `.CodeMirror`.

**Dynamic content**: `buildMenuHTML()` adapts based on:
- `hasSel` — whether text is selected (shows AI Optimize + Cut/Copy)
- `hasImage` — whether an image is right-clicked (shows AI Describe Image + Image Size)
- `caps` — model capabilities (hides vision/web features when unsupported)

**Image resize**: Submenu with 100% / 75% / 50% / 33% / 25% / 10% options, sets `img.style.zoom`.

### 10. UI: Streaming Dialogs (`09-ui-dialogs.js`)

All AI operations use a consistent streaming dialog pattern:
1. `buildPromptDialogHTML()` — creates dialog with input, web search checkbox, optional extra checkboxes
2. `transformToStreaming()` — converts input field to streaming output display, shows Stop button
3. AI response streams via `onChunk` callback into the textarea
4. `stream.showResult()` — shows Confirm/Copy/Cancel buttons
5. `stream.showError()` — shows error message with OK button

### 11. UI: Q&A Dialog (`10-ui-qa.js`)

Special handling for code blocks:
- `getCodeBlockContext()` called on dialog open
- If code block detected: hint displayed, specialized coding prompt, "Replace Code Block" button
- `stripCodeFences()` removes markdown fences from AI response before replacement
- Cursor position saved before dialog opens, restored on confirm for reliable insertion

### 12. UI: Settings Panel (`11-ui-settings.js`)

Sections:
1. **API Provider**: ChatGPT OAuth (connection status) or OpenAI Compatible (URL, Key, Models, Save & Test with live log)
2. **Prompts**: System/User for each feature (Optimize, Optimize with Context, Image, Q&A, Q&A with Context)
3. **Shortcuts**: Shortcut recorder for Q&A key combination

### 13. Styling (`12-styles.js`)

CSS injected via `<style>` element covering:
- Toast notifications (info/success/error/progress variants)
- Context menu (frosted glass, dark mode, submenus, animations)
- Dialogs (prompt, streaming, result)
- Settings panel (sections, inputs, selects, shortcut recorders)
- Dark theme via `@media(prefers-color-scheme: dark)`

## Known Issues

| Issue | Status | Notes |
|-------|--------|-------|
| Typora upgrade overwrite | Known | Re-run `install.sh` after Typora updates |
| Token refresh | Not implemented | Only expiry check + prompt; no automatic refresh-token renewal |
| CM instance stale after dialogs | Mitigated | `reactivateBlockCM()` re-creates CM via simulated click; may not work in all edge cases |
| Rendered HTML block matching | Mitigated | `getHtmlBlockSource()` matches by tag name + text content; may fail if duplicated HTML blocks exist |

## Build, Install, and Debug

### Build
```bash
bash build.sh
# Output: src/typora-ai-edit.js (single IIFE, ~2600 lines)
```

### Install
```bash
sudo bash bin/install.sh
# Restart Typora
```

### Update Plugin
```bash
bash build.sh
sudo cp src/typora-ai-edit.js /Applications/Typora.app/Contents/Resources/TypeMark/ai-edit/typora-ai-edit.js
# Restart Typora
```

### Uninstall
```bash
sudo bash bin/uninstall.sh
```

### Debug
Press `Option+Command+I` in Typora for DevTools. Watch Console for `[AI Edit]` prefixed messages.
